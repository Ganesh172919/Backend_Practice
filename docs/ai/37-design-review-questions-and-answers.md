# 37. Design Review Questions and Answers

## Purpose
This document captures 10 evaluator-style design questions with grounded answers about the current AI backend.

## 1. Why is memory hybrid deterministic plus AI-assisted?
The deterministic layer handles obvious phrases reliably and cheaply, while AI-assisted extraction catches more freeform language. This lowers total dependence on provider availability without giving up flexible extraction.

## 2. Why does room AI use both `aiHistory` and persisted `Message` documents?
`Room.aiHistory` is optimized for future prompting, while `Message` is optimized for chat UX, reactions, edits, and deletion. The split supports two use cases, but it also introduces duplication and drift risk.

## 3. Why is quota in-memory, and what breaks in a multi-instance deploy?
In-memory quota is easy to implement and fast, but it only governs one process. Under multiple instances, users can exceed intended limits by hitting different nodes, and counters reset on restart.

## 4. Why are prompt templates DB-overridable?
DB overrides let admins change model instructions without code deployment. That helps fast iteration, but it needs auditability and stronger rollout controls.

## 5. Why does response storage differ between solo chat and room AI?
Solo chat centers around a private `Conversation` document, so embedding the history is straightforward. Room AI must participate in broader room messaging behavior, so it stores user-visible output in `Message` and prompt state in `Room.aiHistory`.

## 6. Why is attachment handling asymmetric across text, image, and PDF files?
Text and small images can be cheaply converted into prompt-friendly forms. PDF extraction is not implemented in the source build, so PDFs are acknowledged but not semantically parsed, creating weaker context for that file type.

## 7. Why does auto-routing use heuristics instead of measured routing?
The current router is simple and deployable without telemetry infrastructure. It uses prompt length, operation type, and name-pattern preferences, but it is not yet backed by cost, latency, or quality measurements.

## 8. How can insights become stale?
Insights are refreshed at specific events, not continuously. If messages change after the last refresh or if a refresh fails, the stored summary may no longer match current room or conversation state.

## 9. Where can race conditions or duplicate writes happen?
They can occur when multiple AI requests update the same conversation, room, or memory fingerprint close together. Because writes span multiple collections without transactions, one side effect can succeed while another fails or lags.

## 10. How can the system scale safely without changing product semantics?
The safest path is to keep request semantics the same while moving coordination state to Redis, pushing secondary work like insights to queues, improving provider health management, and aligning room and solo orchestration around shared services.
