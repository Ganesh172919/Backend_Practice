The Challenge
Why Do We Need InstaSHAP?
SHAP values are the gold standard for ML explanations — but they're too slow for real-world deployment.
🧠

The Opacity Problem

Deep learning models are powerful but opaque. Regulators, doctors, and users need to know why a model made a specific prediction. SHAP assigns each feature a fair importance score using game theory.

⏳

The Speed Problem

Computing exact Shapley values requires 2d model evaluations for d features. Even approximate permutation SHAP needs hundreds of evaluations per sample — taking seconds to minutes.

⚡

The InstaSHAP Solution

Train an additive neural network under a Shapley-kernel masked objective. After training, each component's output IS the Shapley value. One forward pass = all explanations.


Speed Comparison: Traditional SHAP vs InstaSHAP

InstaSHAP produces equivalent feature attributions 80–1500× faster by amortizing the computation into training.

Core Innovation
How InstaSHAP Works
A 4-stage pipeline that transforms expensive Shapley computation into a single forward pass through an additive model.

InstaSHAP Training Pipeline

Black-Box → Masked Surrogate → InstaSHAP Model → Instant Explanations

Stage 1 — Train Black-Box MLP Prediction

A standard 2-layer MLP [256, 128] is trained on the original labels (MSE for regression, CrossEntropy for classification). This is the model we want to explain. It achieves strong predictive performance but provides no interpretability.

Stage 2 — Train Masked Surrogate Approximation

A mask-aware MLP learns to predict f(x; S) — i.e., what the black-box would output if only features in subset S were visible. Takes concatenation of [x ⊙ mask, mask] as input. Trained with Shapley-kernel-weighted random masks.

Stage 3 — Train InstaSHAP Model Explanation

An additive model (per-feature sub-networks [96, 64]) is trained to match the surrogate's masked outputs. The key: under Shapley-weighted masking, the optimal additive decomposition converges to exact Shapley values. This is Equation 20 from the paper.

Stage 4 — Compare & Validate Verification

InstaSHAP explanations are compared against permutation SHAP baseline. The alignment (MSE, MAE) validates that InstaSHAP faithfully recovers the ground-truth Shapley values — but in milliseconds instead of minutes.

Key Mathematical Insight

// Shapley-Kernel Masked Training Objective (Eq. 20) // The InstaSHAP model is trained to minimize: L = Ex, S~Shapley[ ‖ InstaSHAP(x; S) − Surrogate(x ⊙ expand(S), S) ‖² ] // Where masks S are drawn from the Shapley kernel: w(|S|) = 1 / [ C(d, |S|) × |S| × (d − |S|) ] // This weighting ensures convergence to true Shapley values // because the additive model's optimal solution under this kernel // is provably the unique Shapley decomposition.
Architecture
ML Models Used
Five model types form a complete modeling stack — from opaque prediction to transparent explanation.
🔲

TabularMLP Black-Box

Standard feed-forward network: Linear(→256)→ReLU→Dropout(0.10)→Linear(→128)→ReLU→Dropout→Linear(→out). Loss: MSE / CrossEntropy. Optimizer: AdamW. The model we want to explain.

🎭

MaskedSurrogateMLP Surrogate

Mask-aware MLP taking [x⊙mask, mask] as input. Same [256,128] architecture. Learns f(x;S) — the black-box response under feature masking. Differentiable bridge to InstaSHAP.

📊

GAM-1 / GAM-2 Interpretable

GAM-1: one ComponentMLP [96,64] per feature. GAM-2: adds pairwise interaction components (e.g., hour×workingday). Each component output is directly the feature's contribution.

⚡

InstaSHAPModel Core

Extends GAMModel with masked_forward() for Shapley-kernel training. After training, explain() returns per-feature SHAP attributions in one forward pass. The key innovation.

Questions & Answers
Frequently Asked Questions
Everything you need to know about the project, methodology, and results.

Four models in a pipeline: TabularMLP (2-layer [256,128] black-box), MaskedSurrogateMLP (mask-aware bridge), GAMModel (per-feature ComponentMLPs [96,64]), and InstaSHAPModel (extends GAM with masked training). All use PyTorch with AdamW optimizer and early stopping on validation loss.

The black-box MLP doesn't accept mask inputs — it expects full feature vectors. The surrogate bridges this gap by learning f(x;S): what the black-box would predict if only features in set S were visible. It takes [x ⊙ mask, mask] as input, making it differentiable with respect to the masking pattern.

InstaSHAP produces attributions in a single forward pass (8ms for 32 samples). Permutation SHAP took 12.7s for the same 32 Bike samples — that's 1,587× faster. For Covertype: SHAP 0.6s vs InstaSHAP 0.007s (82× faster).

Binary masks sampled from the Shapley kernel distribution: w(|S|) = 1/[C(d,|S|) × |S| × (d-|S|)]. This is NOT uniform random — it favors small and large subsets where marginal contributions are most informative. Edge masks (all-0s, all-1s) are included at 10% probability for stability.

GAM-1 has only univariate components (one sub-network per feature). GAM-2 adds pairwise interaction components (e.g., hour×workingday). GAM-2 can capture feature interactions that GAM-1 misses, often closing the gap with the black-box. For Covertype: GAM-2 = 69.1% vs GAM-1 = 66.9%.

LIME uses local linear approximations with no theoretical guarantee of matching Shapley values. InstaSHAP provably converges to exact Shapley values under the Shapley kernel weighting — it has a game-theoretic foundation that LIME lacks.

Global seed (42) controls Python's random, NumPy's RandomState, and PyTorch's generators. CuDNN deterministic mode is enabled. Classification splits use stratification. All hyperparameters are externalized in config.yaml. Structured logging saves run traces.

CSV tables: model metrics, paper comparisons, explanation fidelity. PNG plots: training curves, feature importance, shape functions, interaction heatmaps, SHAP vs InstaSHAP alignment. JSON summaries: full experiment metadata. PDF reports: multi-page reproducibility report and one-page summary.

Summary
Key Takeaways
💡

Core Innovation

InstaSHAP makes SHAP explanations instant by training an additive model under Shapley-kernel masking. The optimal solution provably equals Shapley values.

🏗️

4-Stage Pipeline

Black-Box → Surrogate → InstaSHAP → Explanations. These are NOT alternatives — each stage builds on the previous.

✅

Validated Results

Adult Income: 84% accuracy matches paper (84.3%). Explanation MSE as low as 0.012 — near-identical to SHAP. Up to 1587× speedup.

🚀

Real-World Impact

Enables real-time explainable AI for healthcare, finance, and regulatory compliance — domains where slow SHAP is impractical.
